## The Problem

It is easy to use deep learning frameworks for years and still keep the actual optimization mechanics at arm's length. That becomes a limit when debugging custom losses, unstable gradients, or architecture behavior that does not match intuition.

This project was about removing abstraction until every forward value and backward derivative was visible.

## Approach

I started from scalar autodiff to make the dependency graph obvious, then scaled to tensor operations where broadcasting and reduction rules become more subtle. The interesting part was not matrix multiplication itself. It was the chain rule across realistic compositions: activations, losses, and batch operations.

Numerical gradient checks were the guardrail. When a derived gradient disagreed with finite differences, the mismatch forced a more precise understanding of the graph rather than more code.

```python
for node in reversed(topological_order):
    node.backward()
    for parent, grad_fn in node.parents:
        parent.grad += grad_fn(node.grad)
```

The main output is conceptual leverage. Once the mechanics are explicit, debugging optimization behavior becomes far less mystical. That payoff transfers directly to larger model work.

## Key Learnings

- Writing the backward pass by hand forces precision that high-level APIs often hide.
- Numerical checks are one of the fastest ways to build confidence in math-heavy code.
- Framework abstractions are powerful only after the underlying mechanics are trusted.
